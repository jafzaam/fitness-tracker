# Ponytail Skill (Lazy Senior Developer & YAGNI)

ทักษะนี้บังคับใช้หลักการ YAGNI (You Ain't Gonna Need It) เพื่อให้เอเจนต์เขียนโค้ดที่เรียบง่ายที่สุด หลีกเลี่ยงการสร้างฟังก์ชันและ Abstractions ที่เกินความจำเป็น

## กฎการตัดสินใจก่อนลงมือเขียนโค้ด (YAGNI Decision Ladder)
ก่อนจะสร้างโค้ดหรือฟังก์ชันใหม่ เอเจนต์ต้องประเมินตามขั้นบันไดนี้:
1. **งานนี้จำเป็นต้องมีจริงหรือไม่?** (หากไม่จำเป็น ให้หยุด)
2. **มีอยู่ในโค้ดเบสเดิมแล้วหรือไม่?** (ให้พยายามนำของเก่ากลับมาใช้ใหม่)
3. **ใช้ Standard Library จัดการได้หรือไม่?**
4. **ใช้ฟีเจอร์ Native ของ Platform/OS จัดการได้หรือไม่?**
5. **ใช้ Dependency ที่ติดตั้งอยู่แล้วจัดการได้หรือไม่?**
6. **ทำเสร็จในบรรทัดเดียวได้หรือไม่?** (เช่น Single expression)
7. **หากไม่เข้าข่ายข้อใดเลย:** ให้เขียนโค้ดที่สั้นที่สุดและเรียบง่ายที่สุดที่ทำให้งานทำงานได้ (Minimal Viable Code)

## แนวทางปฏิบัติ
- **No Over-engineering:** ห้ามออกแบบระบบเผื่ออนาคต
- **Delete unused code:** ลบโค้ดที่ไม่ใช้งานออกทันที
- **Keep it simple:** ปฏิเสธการเพิ่มความซับซ้อนใดๆ ของโค้ด
