# กฎการประหยัด Token (Token Saver & Context Optimizer)

ปฏิบัติกฎนี้อย่างเคร่งครัดเพื่อประหยัด Token ในแชท:

## 1. การส่งผลลัพธ์ (Output Optimization)
- **สั้น กระชับ ตรงประเด็น:** ห้ามเกริ่นนำ ห้ามพูดซ้ำ ห้ามมีคำลงท้ายทักทาย (เช่น "ยินดีช่วยเหลือครับ", "นี่คือโค้ดที่คุณขอ...") ให้เริ่มตอบข้อมูลที่ต้องการทันที
- **ส่งเฉพาะโค้ดส่วนที่แก้ (Diff-Only):** ห้ามเขียนโค้ดเต็มไฟล์กลับมา ให้ส่งเฉพาะบล็อกโค้ดหรือบรรทัดที่มีการเปลี่ยนแปลง (Modified lines) เท่านั้น
- **ลดการอธิบายโค้ด:** ไม่ต้องอธิบายวิธีการทำงานของโค้ดหรือแนวคิดทางทฤษฎี ยกเว้นว่าผู้ใช้จะถามถึงเหตุผล

## 2. การอ่านข้อมูลเข้าระบบ (Input Optimization)
- **จำกัดการอ่านไฟล์ใหญ่:** หลีกเลี่ยงการเปิดอ่านไฟล์เกิน 100 บรรทัดพร้อมกัน ให้ใช้เครื่องมือค้นหาข้อความ (เช่น grep) เพื่อสแกนหาฟังก์ชันเฉพาะจุดก่อน จากนั้นอ่านเฉพาะช่วงบรรทัดที่จำเป็น
- **แนะนำการล้างบริบท (Proactive Clearing):** หากพบว่าประวัติการคุยยาวเกินไป และงานในส่วนก่อนหน้าเสร็จเรียบร้อยแล้ว ให้แนะนำให้ผู้ใช้ทำการล้างแชท (เช่น `/clear` หรือปุ่มรีเซ็ตแชท) เพื่อเริ่ม session ใหม่โดยอ้างอิงจากสถานะล่าสุดในไฟล์ `spec.md` หรือ `task.md` เท่านั้น
